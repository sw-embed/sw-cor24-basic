Added RUN command to execute stored programs sequentially. Implemented as a run-flag (`mi`) checked in the main loop: when set, the loop copies each stored line's tokens into `tb[]` and calls `dispatch`, advancing a pointer (`tl` reused) until `pe` is reached or an error occurs. Kept the monolith alive by removing the unused ABS builtin and the SAVE/LOAD/PEEK/POKE keyword table entries (freed ~400 bytes; source now 8051 bytes, back under the 8KB UART ceiling). Verified via scripts/run-basic.sh: stored lines execute in order, NEW clears, immediate mode still works after RUN. basic_exec.pas split deferred until the unit system step (018/019).
