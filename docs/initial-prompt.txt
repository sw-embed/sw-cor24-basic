== WHAT THIS PROJECT IS ==

sw-cor24-basic implements "COR24 BASIC v1", a 1970s time-sharing-style,
line-numbered, integer-only BASIC interpreter that runs on the COR24
p-code virtual machine. Inspired by UNIVAC 1100-series terminal BASIC
with teletype and paper-tape workflows.

Key design decisions already locked in (from research.txt):
- Line-numbered stored programs, tokenized on entry
- Integer-only (signed 24-bit), no floating point in v1
- Interactive: immediate mode + stored program mode (RUN)
- Terminal/teletype oriented with paper-tape SAVE/LOAD
- Runs on the p-code VM as a language layer, NOT compiled to p-code
- Three-layer architecture:
  - Layer 1: p-code VM (language-neutral substrate)
  - Layer 2: BASIC runtime (I/O, line storage, stacks, PEEK/POKE)
  - Layer 3: BASIC interpreter (tokenizer, parser, dispatch, execution)
- Tokenized internal representation (keywords, operators, literals)
- Recursive-descent expression parser (Pratt/precedence-climbing)
- Statement dispatcher keyed on first token
- Fixed-size tables and stacks (no GC, no heap in v1)
- Scalar integer variables only (A-Z, optionally A0-Z9)
- String literals for PRINT, no string variables in v1
- Byte-oriented PEEK/POKE, word access (PEEKW/POKEW) later
- Classic error messages (SYNTAX ERROR, WHAT?, etc.)
- No BASIC-specific VM opcodes — only language-neutral VM primitives
- Dialect name: "COR24 BASIC v1" (not "BASIC77", not "Tiny BASIC")

v1 statements: LET, PRINT, INPUT, IF...THEN, GOTO, GOSUB, RETURN,
FOR...TO...STEP, NEXT, STOP, END, REM
v1 commands: LIST, RUN, NEW, SAVE, LOAD, BYE
v1 functions: PEEK, POKE, ABS

== P-CODE / VM / PASCAL ECOSYSTEM ==

BASIC runs on the p-code VM. You MUST understand these sibling repos:

- ~/github/sw-embed/sw-cor24-pcode/
  The p-code VM implementation. This is the execution substrate for
  BASIC. The VM owns the stack/call model, memory model, traps,
  device access. BASIC is an interpreter RUNNING ON the VM, not
  compiled to p-code. Understand the VM's instruction set, memory
  layout, and syscall/trap mechanisms before designing BASIC's
  runtime layer.

- ~/github/sw-embed/sw-cor24-plsw/
  Pascal-to-p-code compiler. Pascal already defines the p-code
  instruction set. BASIC must work within those existing opcodes.
  If interpreter-assisting operations are needed (byte load/store,
  memcpy, indirect jump, generic syscall), they should be proposed
  as language-neutral VM extensions useful to ALL languages, not
  BASIC-specific additions.

- ~/github/sw-embed/sw-cor24-pascal/
  Pascal language support and runtime. Understanding the Pascal
  runtime helps inform how BASIC's runtime should layer on the VM
  without conflicting.

- ~/github/sw-embed/sw-cor24-x-pc-aotc/
  AOT compiler translating p-code to native COR24 assembly. Relevant
  because BASIC's runtime code may eventually benefit from AOT
  compilation of hot paths.

- ~/github/sw-embed/sw-cor24-emulator/
  The COR24 emulator. All testing runs here. Understand how to load
  and run p-code programs on the emulator.
